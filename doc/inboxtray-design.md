## 一、需求背景与痛点

### 1.1 用户场景

在 Windows 端学习和工作时，经常遇到以下情况：

- **临时想法** — 突然想到一个点子、待办小事，但打开 Obsidian 等大型笔记软件太重，打断工作流
- **稍后阅读的网页** — 打开的网页标签太多占内存，收藏到书签需要想分类，后续容易遗忘
- **快速剪藏** — 看到有用的文字、图片、链接，希望快速"丢"到一个地方，稍后再整理

### 1.2 核心诉求

- **快速调出** — 全局快捷键呼出小窗口，5秒内完成记录
- **混合内容** — 文字、链接、图片都可以放进去
- **无需分类** — 先存再说，不中断当前工作
- **后续整理** — 可选推送到 Obsidian 进行深度整理

### 1.3 现有工具的不足

| 工具类型 | 代表                    | 不足                  |
| ---- | --------------------- | ------------------- |
| 稍后阅读 | Pocket、Cubox、Raindrop | 偏重阅读，不适合记临时想法，打开不够快 |
| 快速记录 | Windows 便笺、Flomo      | 只能文字，不能存链接，便笺无分类    |
| 剪贴板  | Ditto                 | 只是剪贴板历史，不是主动记录工具    |
| 笔记软件 | Obsidian、Notion       | 打开慢，不适合"临时丢进去"的场景   |

**市场空白：没有一个轻量级的"桌面临时收件篮"工具。**

***

## 二、产品定位

**InboxTray = 桌面临时收件篮**

- 定位：介于"剪贴板"和"笔记软件"之间的快速捕获工具
- 理念：先收集，再处理（GTD 收集阶段）
- 关系：暂存箱是 Obsidian 的"快速入口"，不是替代品

```
工作中临时想到/看到 ──▶ 暂存箱（5秒内完成）
                              │
                              ▼
                    后续整理时间 ──▶ 推送到 Obsidian Inbox/
                              │
            ┌─────────────────┼─────────────────┐
            ▼                 ▼                 ▼
        知识库(文章)      项目管理(任务)      日常记录(想法)
```

#### 交互细节

- 呼出窗口后焦点自动在输入框（延迟 100ms 确保窗口渲染完成）
- 输入完成后按 `Ctrl+Enter` 保存，`Esc` 关闭窗口
- 窗口关闭后应用继续在托盘运行
- 支持文本中内嵌 URL，系统会自动提取并抓取页面元数据
- 纯链接类型：内容自动填充为页面 og:description
- 窗口可拖拽边缘临时调整大小，关闭后恢复默认尺寸

### 4.3 数据库设计

#### 主表：items

> **重要变更**：表名从 `inbox` 改为 `items`，字段名也有调整。实际使用 `content_type` 字段存储类型，而非通过字段判空推断。

```sql
CREATE TABLE items (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    content     TEXT    NOT NULL DEFAULT '',   -- 用户输入的文字内容
    content_type TEXT   NOT NULL DEFAULT 'text', -- 'text' | 'link' | 'image' | 'mixed'
    title       TEXT    DEFAULT '',            -- 页面自动抓取的标题
    url         TEXT    DEFAULT '',            -- 链接 URL（如果有）
    image_path  TEXT    DEFAULT '',            -- 图片文件名，多张用 ; 分隔
    status      TEXT    NOT NULL DEFAULT 'pending',  -- 'pending' | 'pushed' | 'done'
    created_at  INTEGER NOT NULL,              -- Unix 时间戳（毫秒）
    pushed_at   INTEGER DEFAULT 0              -- 推送时间戳
);

-- 索引
CREATE INDEX idx_items_status ON items(status);
CREATE INDEX idx_items_created ON items(created_at DESC);
```

#### 配置表：settings

```sql
CREATE TABLE settings (
    id INTEGER PRIMARY KEY CHECK (id = 1),
    hotkey TEXT NOT NULL DEFAULT 'Ctrl+Alt+Q',
    obsidian_path TEXT DEFAULT '',
    inbox_folder TEXT NOT NULL DEFAULT 'Inbox',
    theme TEXT NOT NULL DEFAULT 'brass',
    auto_start INTEGER NOT NULL DEFAULT 0
);
```

#### 字段设计说明

实际实现使用显式的 `content_type` 字段，而非设计初期设想的"通过字段判空推断类型"。类型定义：

| content\_type | 说明                        |
| ------------- | ------------------------- |
| `text`        | 纯文字                       |
| `link`        | 纯链接（内容可能为 og:description） |
| `image`       | 纯图片                       |
| `mixed`       | 文字 + 图片混合                 |

**显示时自动判断类型标签：**

```typescript
function getDisplayType(item: Item): string {
    if (item.url && item.imagePath && item.content) return '图文链接';
    if (item.url && item.imagePath) return '链接+截图';
    if (item.url && item.content)     return '笔记+链接';
    if (item.imagePath && item.content) return '图文笔记';
    if (item.imagePath)              return '图片';
    if (item.url)                     return '链接';
    return '文字';
}
```

#### 数据量评估

- 每天记录 10 条，一年 3650 条
- 每条平均 200 字，数据库约 5-10 MB
- 三年不清理约 20-30 MB，SQLite 完全够用

### 4.4 文件存储结构

```
AppData/Local/com.inbox-tray.app/
├── inboxtray.db         -- SQLite 数据库
├── inboxtray.log        -- 应用日志（由 Rust 端写入）
├── settings.json        -- 用户配置（通过 tauri-plugin-store 管理）
├── image_1234567890.png -- 图片附件（命名格式：image_{timestamp}.png）
```

> **变更说明**：图片直接存储在 app\_local\_data 根目录下，文件名格式为 `image_{timestamp}.png`，多张图片通过 `;` 分隔存储在 `image_path` 字段中。删除条目时自动同步删除关联本地图片。

## 五、核心流程设计

### 5.1 快速记录流程

```
用户按全局快捷键
       │
       ▼
  快速输入窗口弹出（焦点在输入框，延迟 100ms）
       │
       ├── 输入文字 → 按 Ctrl+Enter → content='文字内容', content_type='text'
       │
       ├── 粘贴 URL → 自动识别 → 后台抓取 title 和 og:description
       │   ├── content_type='link'
       │   ├── title='页面标题'
       │   ├── url='链接地址'
       │   └── content='og:description 或 URL 本身'
       │
       ├── 粘贴图片（含 Snipaste 截图后粘贴）→ image_path='图片文件名'
       │   └── 用户可继续输入文字说明 → content='文字说明', content_type='mixed'
       │
       └── 混合场景：粘贴图片 + 输入文字 + 粘贴链接 → content_type='mixed'
       │
       ▼
  窗口隐藏（应用继续在托盘运行）
```

> **变更说明**：
>
> - 保存快捷键从 `Enter` 改为 `Ctrl+Enter`
> - 抓取元数据不仅获取 title，还获取 og:description
> - 纯链接的内容自动填充为 og:description
> - 支持 Snipaste 截图粘贴（剪贴板类型为 Files）

### 5.2 推送到 Obsidian 流程

```
用户在列表中点击单条内容的"推送"按钮
       │
       ▼
  调用 Rust 命令 push_to_obsidian
       │
       ├── 检查 obsidian_path 是否已配置
       │
       ├── 创建 {vault_path}/{inbox_folder}/ 目录（含 attachments/ 子目录）
       │
       ├── 复制 AppLocalData 中的图片到 inbox/attachments/
       │
       ├── 生成 Markdown 文件（文件名：{timestamp}_{摘要}.md）
       │
       └── 写入目标目录
       │
       ▼
  更新数据库状态为 'pushed'
       │
       ▼
  列表刷新，显示已推送状态
```

> **变更说明**：当前仅支持单条推送，批量推送（勾选多条）尚未实现。

### 6.4 数据隐私声明

应用所有数据（SQLite 数据库、图片附件、用户配置）均存储在本地，**不会上传至任何服务器**。具体存储路径：

```

Windows: C:\Users{用户名}\AppData\Local\com.inbox-tray.app\\

```

用户可通过设置页"关于"板块查看当前数据存储位置。推送至 Obsidian 时，数据仅写入用户指定的本地 Obsidian 仓库目录。

## 七、与 Obsidian 的关系设计

### 7.1 方案：可选推送（方案 C）

- 暂存箱是独立系统，数据存在本地 SQLite
- 推送到 Obsidian 是"复制一份过去"，暂存箱保留记录
- 推送后状态标记为 `pushed`，不会重复推送
- 暂存箱内容生命周期：录入 → 暂存 → \[可选]推送 → 标记完成/删除

### 7.2 推送目标

- 统一推送到 Obsidian 仓库的 `Inbox/` 文件夹
- 用户在 Obsidian 中自行决定每个文件最终去向
- 符合 GTD "收件篮"理念

### 7.3 推送模式

| 阶段       | 推送模式     | 说明                 |
| -------- | -------- | ------------------ |
| MVP（第一期） | 单条推送     | 每条内容生成一个独立 .md 文件  |
| 后续（第二期）  | 合并推送（可选） | 勾选多条内容合并为一个 .md 文件 |

## 八、技术实现要点

### 8.1 全局快捷键

- 使用 Tauri v2 的 `tauri-plugin-global-shortcut` 插件
- 默认 `Ctrl+Alt+Q`，用户可在设置中修改
- 快捷键冲突处理：注册失败时在前端显示错误提示
- 使用全局 `Mutex<Option<Shortcut>>` 共享当前快捷键状态

### 8.2 链接标题抓取

```

用户粘贴 URL（或文本中包含 URL）
│
▼
正则提取第一个 URL
│
▼
发送 HTTP GET 请求（超时 10 秒）
│
├── 成功 → 解析 HTML
│   ├── <title> → 存入 title 字段（过滤垃圾标题）
│   └── og:description → 存入 content 字段（纯链接时）
│
└── 失败/超时 → title 留空，content 显示 URL 本身

```

> **变更说明**：
>
> - 超时从 3 秒增加到 10 秒
> - 同时抓取 title 和 og:description
> - 增加了标题有效性过滤（排除 404、error 等垃圾标题）

### 8.3 图片处理

- 粘贴板图片：读取 `clipboardData.items` 中的图片数据，或使用 `readImage()` 读取 Snipaste 截图
- 图片转换为 PNG 格式后保存
- 文件名格式：`image_{timestamp}.png`
- 图片保存到 `AppData/Local/com.inbox-tray.app/`
- 单条记录最多支持 5 张图片，多张以 `;` 分隔

> **变更说明**：
>
> - 支持 Snipaste 截图（剪贴板类型为 Files，需用 `readImage()` API）
> - 图片通过 `tauri-plugin-fs` 写入 AppLocalData
> - 前端使用 `convertFileSrc` + `appLocalDataDir()` 显示本地图片
> - **粘贴图片时自动压缩**：最大尺寸 1920px 等比缩放，最大文件大小 2MB（通过 canvas JPEG 压缩）

### 8.4 推送到 Obsidian

- 读取配置中的 obsidian\_path 和 inbox\_folder
- 检查目标目录是否存在，不存在则创建
- 生成 Markdown 文件，文件名格式：`{YYYY-MM-DDTHH-MM-SS}_{摘要}.md`
- 同名文件自动去重（追加 `-1`、`-2`...）
- 图片复制到 `inbox/attachments/` 目录
- Markdown 中图片引用使用 Obsidian Wikilink 语法：`![[attachments/xxx.png]]`
- 更新数据库状态

### 8.5 系统托盘

- 关闭窗口时不退出应用，最小化到系统托盘
- 托盘图标右键菜单：显示 InboxTray / 退出
- 左键单击托盘图标也可打开主窗口
- 开机自启动：启动时自动注册 Windows 开机启动项

### 8.6 主题系统

- 使用 CSS 变量 + `data-theme` 属性切换主题
- 四种预设主题：brass（黄铜暗色）、light（亮色）、slate（石板）、forest（森林）
- 主题配置持久化到 settings.json
- 快速输入窗口通过 `theme-changed` 事件同步主题变更

### 8.7 复选框交互

- 列表中 `- [ ] `  / `- [x] `  样式的文本渲染为可点击复选框
- 仅 `pending` 状态的条目支持点击切换
- 点击后更新数据库中的 content 字段

## 十、风险与注意事项

| 风险              | 应对措施                                                   |
| --------------- | ------------------------------------------------------ |
| 全局快捷键冲突         | 注册失败时在前端提示用户修改                                         |
| 链接标题抓取失败        | 设置 10 秒超时，失败时显示 URL 本身                                 |
| Obsidian 仓库路径变更 | 推送前校验路径是否存在                                            |
| SQLite 数据损坏     | 启动时校验数据库完整性，损坏时提示重置                                    |
| 图片占用空间过大        | 单条限制 5 张图片，自动压缩（1920px/2MB），删除条目时同步清理图片                |
| 安装路径权限问题        | NSIS 安装器以用户权限运行，避免写入系统目录                               |
| 异常崩溃无日志         | Rust panic 捕获写入日志文件，前端全局 Error Boundary                |
| 数据库损坏           | 启动时自动执行 PRAGMA integrity\_check，结果写入日志                 |
| 安装后数据残留         | 卸载时仅删除安装目录，数据目录保留在 AppData（符合 Windows 规范），用户可选是否同步删除数据 |
